Detailed Description

The Session class reflects the ongoing state of the session, not just a snapshot. The information available includes access token, refresh token and id tokens. Additionally authorization state from AppAuth library, id token payload, scopes, session status and online status. The session object can be acquired by calling SessionManager#getSession()

SessionStatus org.vaultit.mobilesso.mobilessosdk.Session.getStatus ( )

Determines session status. If value returned is VALID, then tokens are in usable state, ie. refreshToken is valid and can be used to refresh the access token. If value returned is EXPIRED, then refresh token is expired, but browser's SSO session cookie might still be valid, in which case a new authentication is not needed. This can be checked by performing a "session check", which means calling SessionManager#authorize() with prompt=none.

Returns
returns one of: VALID | EXPIRED | NO_SESSION

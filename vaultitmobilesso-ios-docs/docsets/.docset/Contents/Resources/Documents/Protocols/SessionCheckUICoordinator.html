SessionCheckUICoordinator

@objc public protocol SessionCheckUICoordinator: NSObjectProtocol

A public protocol to allow customizing the UI effort needed to perform the Safari-based session check.

  • willBeginSessionCheck(completion:)

    This method will be called before any other UI action will take place. Here you can decide how to inform the user about the Safari that is about to pop into the screen. The check will not continue until you call the provided completion block.

    Parameter completion: Call this function block when you are ready to continue the session checking process.

    Declaration

    Swift

    func willBeginSessionCheck(completion: @escaping () -> Void)
  • willDownloadDiscoveryDocument()

    This method will be called when the SDK will begin downloading the GLUU discovery document to find out all the required login authorization endpoints and other configuration. This might be a very fast or very slow process depending on the network connection. Here you can show some sort of spinner or something.

    Declaration

    Swift

    func willDownloadDiscoveryDocument()
  • willPresentSafariViewController()

    This method is called when the discovery doc is downloaded and before the Safari is presented.

    Declaration

    Swift

    func willPresentSafariViewController()
  • didFinishWithResult(success:completion:)

    This method is called when the session check finishes. The success parameter is true if a session was found, false otherwise. The SessionManagerDelegate will only be called after you call the provided completion block.

    Declaration

    Swift

    func didFinishWithResult(success: Bool, completion: @escaping () -> Void)

    Parameters

    completion

    Call this function block when you are ready to provide the result with the SessionManagerDelegate.

SessionManager

open class SessionManager

The main class for handling session status. The class follows a normal singleton pattern, so you should use it via the SessionManager.shared static instance.

  • presentLogin(in:extraScopes:acrValues:stateCompletion:completion:)

    Present the browser that will allow the user to log in and create a session.

    NOTE: This method can also be used for step-up authentication. If you present a login again with a changed acrValues parameter, the new login prompt will be displayed and the current session is replaced.

    Declaration

    Swift

    open func presentLogin(in viewController: UIViewController, extraScopes: [String] = [], acrValues: [String] = [], stateCompletion: ((LoginState) -> Void)? = nil, completion: SessionCompletionListener? = nil)

    Parameters

    viewController

    The host UIViewController that will present the Safari browser.
    extraScopes

    You should set this to all scopes you need beyond the basic OpenID and Profile scopes, which are automatically used.
    acrValues

    This parameter is directly passed as the “acr_values” query parameter as described in the OpenID Connect specification. The “acr” is an abbreviation of “Authentication Context Class Reference”, and is used to give authentication hints to the authorizing server.
    stateCompletion An optional callback that is called when intermediate login states complete

    . This is useful to display an indication to the user. Any errors while loading are supplied normally with the completion callback.
    completion

    The completion block. Will be called with the established session or an error if the login did not succeed.

  • logout(in:loadingCompletion:completion:)

    Logout the user. This method will first check if the session is still valid and then will open a logout URL in a SFSafariViewController if it was. All references to the Session model will be invalid after a succesful logout.

    Declaration

    Swift

    open func logout(in viewController: UIViewController, loadingCompletion: (() -> Void)? = nil, completion: ((_ error: Error?) -> Void)? = nil)

    Parameters

    viewController

    The UIViewController that will present the Safari browser.
    loadingCompletion An optional callback that is called when the auth configuration is loaded from the issuer and before the

    SFSafariViewController will be presented. This is useful to display a loading indication to the user before the Safari can be presented. Any errors while loading are supplied normally with the completion callback.
    completion

    The completion listener. Will return an error if something happened. See SessionManagerErrorCode enum for possible error codes.
